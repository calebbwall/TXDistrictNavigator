Objective
Update the existing uploaded codebase (Expo + Express + Drizzle) so elected officials’ public data stays up-to-date, refreshed weekly, while guaranteeing that refreshes never overwrite any user-entered private data (notes, personal phones/addresses, spouse/kids, birthdays, etc.). Implement robust parsing and fail-safe validation so markup changes do not corrupt the dataset.

Do not rewrite UI/screens/navigation/theme. Only change backend + data fetching + minimal client wiring to replace mock data with API reads.

1) Data model: strict public/private separation

Extend Drizzle schema with two tables:

official_public (refreshable)

id UUID PK

source_member_id string (stable ID from source)

chamber string

full_name string

party string nullable

photo_url string nullable

capitol_address string nullable

capitol_phone string nullable

district_addresses JSON nullable

district_phones JSON nullable

website string nullable

email string nullable

Unique constraint: (source, source_member_id)

official_private (user-entered only)

personal_phone, personal_address

spouse_name

birthday, anniversary

Hard rule: refresh code may ONLY write to official_public. Never touch official_private.

Soft deactivation: if a member disappears from the official source, set official_public.active=false. Never delete.

2) Refresh pipeline: weekly + robust + fail-safe

Create server/jobs/refreshOfficials.ts that refreshes 3 datasets.

A) TLO House refresh (robust scraping)

Fetch list page and extract member detail URLs using tolerant logic (avoid fragile CSS class dependence).

For each member detail page, extract fields using label-based parsing + fallbacks:

Prefer matching text labels like “District”, “Capitol Address”, “Capitol Phone”, “District Office”.

Implement fallback extraction paths:

primary selector

alternate selector

label-adjacent text scan

Derive source_member_id from stable member code or detail-page URL.

Validate record before upsert:

name non-empty

district numeric in 1–150

at least one contact field present (address or phone)
If a single record fails validation, skip it and log warning (do not fail entire job).

Upsert into official_public with source="TX_HOUSE".

B) TLO Senate refresh

Same as House, but validate district numeric in 1–31. Upsert with source="TX_SENATE".

C) U.S. House (Texas delegation) via Congress.gov API

Use CONGRESS_API_KEY and fetch House members from Congress.gov API, filtering to Texas delegation (state=TX). Map fields conservatively (null if missing). Use Congress.gov member identifier as source_member_id. Upsert with source="US_HOUSE".

D) Fail-safe dataset sanity checks (critical)

After parsing, before committing writes:

Compute roster counts for each dataset.

Compare to last successful refresh counts (store these counts + timestamp).

If any dataset deviates by >25% (or is zero), treat as suspicious:

Abort DB writes

Log error with details

Preserve last-known-good dataset

Always log:

parsed count / upserted count / skipped count (validation failures)

deactivated count

runtime duration
