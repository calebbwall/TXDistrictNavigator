Replit Continuous Agent Prompt — Fix Android “random symbol” icons everywhere WITHOUT breaking Expo Go

Problem

On Android Expo Go, many icons (especially navigation icons) render as random symbols. iOS looks correct. The last attempted fix caused Expo Go to stop running, so we need a safe, Expo-managed approach.

Goal
	1.	Make all icons across the app (tabs, headers, buttons, cards, toggles) render correctly on Android and iOS, matching iOS.
	2.	Fix it without ejecting, without native linking, and without adding non-Expo-safe dependencies.
	3.	Ensure the app still launches in Expo Go after the change.

⸻

Hard Rules (do not violate)
	•	Do NOT add or require any native linking steps.
	•	Do NOT use react-native-vector-icons directly.
	•	Do NOT add custom .ttf icon fonts or createIconSet.
	•	Use Expo-compatible installs only: expo install, not npm install, for Expo SDK deps.
	•	If font loading is added, it must be done in a minimal, safe way that cannot deadlock app startup.

⸻

Step 1 — Standardize all icons to Expo-safe components

1.1 Audit + replace icon usage globally

Search for and replace any icon usage that is not Expo-safe:
	•	Replace imports from:
	•	react-native-vector-icons/*
	•	any custom Icon components that set fontFamily manually
	•	any icon library that relies on bundled fonts not loaded via Expo

Standardize to only:
	•	@expo/vector-icons (Ionicons / MaterialIcons / Feather / etc.)

Update every file that renders icons:
	•	bottom tab navigator icons
	•	header icons
	•	all custom buttons with icons
	•	any cards that show icons
	•	settings toggles icons

Deliverable: No remaining imports from react-native-vector-icons anywhere.

⸻

Step 2 — Add one centralized Icon wrapper (prevents regression)

Create src/components/AppIcon.tsx (or similar) that:
	•	only uses @expo/vector-icons
	•	provides a unified API: name, size, color, family
	•	defaults to one family (e.g., Ionicons) unless specified

Then update all code to use AppIcon instead of direct icon imports.

Acceptance: There is exactly one place icons are chosen/implemented.

⸻

Step 3 — Fix fonts the Expo-safe way (avoid crashes)

3.1 Minimal font loading, no deadlocks

In App.tsx (or the actual Expo root):
	•	Use expo-font and useFonts
	•	Load the icon fonts required by the families used in the app.
	•	Render navigation only after fontsLoaded === true
	•	While loading, render a simple <View /> placeholder (no navigation)

Important: do NOT depend on deprecated splash APIs or anything that can hang startup.

3.2 Only load what you use

If you only use Ionicons + MaterialIcons, load only those fonts.
Do not load every icon font.

⸻

Step 4 — Ensure installs are Expo-managed

Run:
	•	expo install expo-font @expo/vector-icons

Do not add or change native config files.

⸻

Step 5 — Prevent the “Expo Go won’t run” regression

Before committing:
	•	Start the app locally (or in Replit) and confirm it boots in Expo Go.
	•	If the app fails to start, revert to the last known good commit and apply changes incrementally:
	1.	Replace icon imports only (no font loading yet)
	2.	Confirm Expo Go boots
	3.	Add minimal font loading
	4.	Confirm Expo Go boots again

⸻

Step 6 — Clear caches and retest on Android

After changes:
	•	Restart Metro bundler with cache cleared (expo start -c equivalent)
	•	Reload Android Expo Go
	•	Confirm:
	•	Tab icons correct
	•	All button icons correct
	•	No random glyphs anywhere

⸻

Acceptance Criteria
	•	App launches normally in Expo Go on Android.
	•	All navigation icons match iOS.
	•	All button icons match iOS.
	•	No random symbols/glyphs remain anywhere.
	•	No flicker from icons loading late (navigation must wait for fontsLoaded).

Deliverables to include in your response
	•	List of all files changed
	•	Confirmation there are zero react-native-vector-icons imports remaining
	•	Which icon families are used and which fonts are loaded
	•	The new AppIcon wrapper location and example usage
