Yep — I looked through what you uploaded, and the core issue is exactly the thing I was hinting at:

What’s happening (based on your actual code)

Your navigation icons (and lots of other icons) are rendered with:
	•	Feather from @expo/vector-icons

Example (your file): MainTabNavigator.tsx uses:

import { Feather } from "@expo/vector-icons";
...
<Feather name="map" size={size} color={color} />

That Feather component ultimately renders a glyph character inside a <Text> using the Feather icon font. If the Feather font isn’t loaded, Android shows random symbols (because it falls back to a different font and those “private use” glyphs become garbage).

✅ In your upload, you are not loading icon fonts anywhere (App.tsx just mounts navigation immediately).
That’s why iOS can look fine (it sometimes masks the problem), while Android looks cursed.

So: the problem is not your icon names; it’s font loading.

⸻

Why your last prompt may have “stopped Expo Go from running”

The most common failure mode is:
	•	the agent added useFonts() but didn’t install expo-font properly, or
	•	the app returned null forever because fontsLoaded never becomes true (bad import / wrong hook usage), or
	•	it introduced a non-Expo-safe icon dependency.
