F5 — Offline-first polish (foundation)

F5.1 Offline detection + UX
	•	Detect online/offline status in React Native.
	•	When offline:
	•	show a subtle banner: “Offline — showing saved data”
	•	disable actions that require network (e.g., place lookup) with clear messaging
	•	When back online:
	•	auto-dismiss banner and allow actions again

F5.2 Cache last-known-good public data locally

Cache these locally (read-only cache):
	•	officials list(s) (public data) used by Browse and Profile
	•	last refresh timestamp + counts by source
	•	last-used map overlay geojson is already cached; keep it stable

Requirements:
	•	On app start, load cached data first for instant UI.
	•	Then attempt network fetch; on success update cache.
	•	If network fetch fails, keep cached data and show the offline/failed banner (do not blank the UI).

Implementation details:
	•	Use AsyncStorage (or existing storage layer).
	•	Cache keys should include versioning, e.g. cache:v1:officials:ALL.

F5.3 Avoid cache poisoning
	•	If a fetch returns suspicious counts (0 or huge drop), treat as failure and keep last-known-good (reuse Phase E guardrails if present).
	•	Never overwrite cache with obviously bad data.
