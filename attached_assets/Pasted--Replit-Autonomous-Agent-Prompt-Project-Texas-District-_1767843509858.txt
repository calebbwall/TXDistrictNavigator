🧠 Core Data Models (Do Not Collapse)
districts
•	id (PK)
•	district_type ENUM: senate | house | congress
•	district_number INT
•	name TEXT NULL
•	geometry GEOMETRY(MultiPolygon, 4326)
•	effective_start DATE
•	effective_end DATE NULL
•	source_url TEXT
•	updated_at TIMESTAMP
officials
•	id (PK)
•	full_name TEXT
•	office_type ENUM: tx_senate | tx_house | us_house
•	district_id FK → districts.id
•	photo_url TEXT
•	public_json JSONB
•	private_json JSONB (never overwritten by sync)
•	updated_at TIMESTAMP
offices
•	id (PK)
•	official_id FK
•	office_kind ENUM: capitol | district
•	address TEXT
•	phone TEXT
staff
•	id (PK)
•	official_id FK
•	name TEXT
•	role TEXT
Indexes required:
•	GIST index on districts.geometry
•	B-tree on (district_type, district_number)
•	Text index on officials.full_name

Overlay Selection Requirement (Critical)
Users must be able to select none, one, two, or three overlays independently:
•	TX Senate
•	TX House
•	US Congress
UI
•	Three independent toggles
•	Default: ALL OFF
•	Toggling one must not affect the others
Rendering
•	Each overlay is its own Mapbox layer set
•	No overlays selected = no district polygons visible
•	Any combination of overlays must render together
Highlighting

•	Tapping a district highlights only that district within its overlay
•	Toggling OFF an overlay must not break others
Backend

•	GET /districts?type=congress
