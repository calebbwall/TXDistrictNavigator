Replit Continuous Agent Prompt

Stop the loop: Fix iOS WebView NSURLErrorDomain -1004 for Map screen with proof

Problem
On iOS (Expo Go), the Map screen WebView repeatedly shows:
	•	“Error loading page”
	•	NSURLErrorDomain
	•	Error Code: -1004
	•	“Could not connect to the server”

This persists despite prior “fixes.” You must assume prior changes were incorrect or not running. I need a fix that works on-device, not just in browser preview.

Non-negotiable constraints
	1.	The WebView must NOT load any URL that depends on device reaching “localhost”, “127.0.0.1”, “0.0.0.0”, or a relative path like /map.html or /api/... from inside the WebView.
	2.	Do NOT use source={{ uri: ... }} unless the URI is fully proven reachable from the phone.
	3.	Preferred architecture: inline HTML for Leaflet in WebView + React Native performs all network fetches + send GeoJSON into WebView via postMessage.
	4.	You must add instrumentation so the app shows exactly what it’s trying to load/fetch.

Step 2 — Instrument WebView load source and errors (must be visible)

Add a dev-only debug panel on the Map screen that displays:

Step 3 — Implement the correct fix (inline HTML, no remote page load)

Change the WebView to only load inline HTML:

<WebView source={{ html: MAP_HTML }} ... />

MAP_HTML must be a complete HTML string that includes Leaflet JS/CSS via CDN or embedded, and creates a map container. This ensures the WebView can load even if the backend is down.

Do NOT load landing-page.html or any server-hosted html file via URI.

In React Native code (MapScreen or helper):

	•	/api/geojson/tx_house
	•	/api/geojson/tx_senate
	•	/api/geojson/us_congress
	•	Show in the debug panel for each overlay:
	•	request URL
	•	HTTP status
	•	bytes

	•	error message if any

IMPORTANT: Do not hardcode localhost.
Use an existing API base helper if present. If not, implement a robust one for Replit + Expo Go:
	•	If you have a published URL for the server, use that.
	•	Otherwise derive host from Expo Constants and append the backend port, and display it in debug panel.

Step 5 — WebView must only receive data, never fetch it

WebView creates and toggles Leaflet layers from provided GeoJSON. No network calls from WebView.

Why this fixes it

NSURLErrorDomain -1004 is the WebView failing to load a URL. Inline HTML removes that failure class entirely. RN-side fetches are far more reliable in Expo Go than WebView-relative fetches.
