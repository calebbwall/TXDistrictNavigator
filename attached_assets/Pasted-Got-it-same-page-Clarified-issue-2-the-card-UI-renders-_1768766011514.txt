Clarified issue #2: the card UI renders, but the fields inside are empty (no name/district/party/photo/etc.). So this is not a “list doesn’t render” problem; it’s a data-shape / data-binding / fetch result problem.

That usually comes from one of these:
	•	API returns data but with different field names than OfficialCard expects (e.g., FullName vs fullName)
	•	API returns only IDs, and the client never does the follow-up fetch
	•	The client is reading from the wrong property (e.g., official.name when the object is { full_name: ... })
	•	The data is still “pending,” but the UI renders placeholder cards without showing a loading state
	•	Serialization bug: data is nested ({ official: {...} }) and the card expects flat
