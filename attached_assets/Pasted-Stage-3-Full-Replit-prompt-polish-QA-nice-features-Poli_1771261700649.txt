Stage 3 — Full Replit prompt (polish + QA + “nice” features)

Polish, harden, and refine the Prayers feature. Stage 2 UI is implemented. This stage focuses on: visual clarity, UX friction removal, reliability, edge cases, and small “delight” features that don’t add complexity risk.

Goals
	1.	Make the UI feel calm, fast, and consistent.
	2.	Improve “rhythm” features: rotation fairness, yesterday-exclusion, pinned behavior, needs-attention, streak UX.
	3.	Add “Recently Answered” quality (reflection note display, quick reopen).
	4.	Add optional gentle reminder (local notification) if Expo/React Native supports it in your app build.
	5.	Add safe, useful guardrails: undo, confirmations, empty-state messaging.
	6.	Add lightweight tests and a diagnostics screen to avoid regressions.

B) Rotation & “Today’s 3” Improvements (Algorithm hardening)

Requirements
	1.	Never reshuffle within a day (already).
	2.	Exclude yesterday’s picks unless pool too small.
	3.	PinnedDaily behavior:
	•	If pinnedDaily >= 3, choose 3 pinned with oldest lastShownAt.
	•	If pinnedDaily < 3, fill remaining with non-pinned.
	4.	Add a “no-repeat within 3 days” preference:
	•	Track recent dateKeys in memory by reading last 2 days of picks (if present).
	•	Prefer prayers not shown in last 3 days.
	•	If not enough, relax constraint gradually.
	5.	Add “Needs Attention” scoring:
	•	A prayer needs attention if lastPrayedAt is null or older than X days (default 14).
	•	In “Needs Attention” panel, show oldest first.
	6.	Add a small endpoint enhancement:
	•	GET /api/daily-prayer-picks?forceRegenerate=true (dev-only behind a flag) for testing.

Data updates
	•	When user taps “Mark Prayed” in Focused Mode:
	•	Set lastPrayedAt=now.
	•	Optionally also set lastPrayedAt when user marks “Today Complete”:
	•	Only for prayers in Today’s 3 deck, if they weren’t already updated today.
